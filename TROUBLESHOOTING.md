# 시험장 비상 대응 가이드 (TROUBLESHOOTING)

시험 중 오류가 나면 **에러 메시지의 코드(C..., LNK...)를 보고 아래에서 찾아** 그대로 조치하면 된다.
위에서부터 순서대로: **컴파일 에러 → 링크 에러 → 실행 중 오류 → 결과가 이상함 → 모듈별 실수 → 최후의 수단**

> C 문법 자체가 헷갈리면(구조체·포인터·malloc·파일 입출력) → **C_기초문법.md**
> 함수 사용법/조립 레시피 → **README.md**

---

## 1. 컴파일 에러 (빌드가 안 됨)

### C4996: 'scanf'/'fopen' ... may be unsafe
- **원인**: VS가 표준 함수를 보안 경고로 막음
- **해결**: 해당 .c 파일 **맨 첫 줄**(모든 #include보다 위!)에 추가
  ```c
  #define _CRT_SECURE_NO_WARNINGS
  ```
- 이 오픈소스의 모든 .c에는 이미 들어 있음. **새로 만든 파일**에 빠뜨리기 쉬움.

### C2065: '...' 선언되지 않은 식별자
- 함수/변수 이름 오타이거나 **#include 누락**
- 그 함수가 들어있는 헤더를 include 했는지 확인 (예: `bstInsert` → `#include "bst.h"`)

### C2143 / C2059: 구문 오류
- 대부분 **바로 윗줄의 세미콜론(;) 누락** 또는 괄호 짝 안 맞음
- 에러 줄이 아니라 **그 위 1~3줄**을 먼저 볼 것

### C1083: 포함 파일을 열 수 없습니다 'xxx.h'
- 헤더 파일이 프로젝트 폴더에 없음 → .h 파일도 같은 폴더에 복사했는지 확인
- 파일명 대소문자/오타 확인 (`cQueue.h` vs `cqueue.h`는 보통 통과되지만 오타는 안 됨)

### 한글 주석 때문에 경고/에러 (C4819 등)
- 파일 인코딩 문제. 이 오픈소스는 **UTF-8(BOM)** 으로 저장되어 있어 정상 컴파일됨
- 직접 만든 파일에서 나면: VS에서 파일 → 다른 이름으로 저장 → 저장 버튼 옆 ▼ → "인코딩하여 저장" → **UTF-8(서명 있음)** 선택
- 급하면: 한글 주석을 지우면 무조건 해결됨

## 2. 링크 에러 (LNK로 시작)

### LNK2019 / LNK2001: 확인할 수 없는 외부 참조 (미해결 외부 기호)
- **원인 1 (제일 흔함)**: 함수가 구현된 **.c 파일을 프로젝트에 추가 안 함**
  → 솔루션 탐색기에서 소스 파일 우클릭 → 추가 → 기존 항목 → 해당 .c 추가
- **원인 2**: 함수 선언만 있고 정의(몸체)가 없음 → 함수 이름 오타 확인

### LNK1169 / LNK2005: 이미 정의되어 있습니다 (main 중복 등)
- **main 함수가 2개 이상**임 (예: main.c와 교재 예제 ex파일을 같이 넣은 경우)
- 안 쓰는 파일을 우클릭 → **"프로젝트에서 제외"** (지우지 말고 제외만)

## 3. 실행 중 오류

### 콘솔 창이 바로 꺼짐
- `Ctrl + F5` (디버그하지 않고 시작)로 실행
- 또는 main 끝 `return 0;` 앞에 `getchar();` 추가

### 파일을 못 읽음 / "[오류] ... 파일을 열 수 없습니다"
- **VS에서 실행하면 작업 폴더는 .vcxproj가 있는 프로젝트 폴더**다. txt 파일이 거기 있는지 확인
- 프로그램이 만든 파일도 그 폴더에 생긴다 (탐색기에서 프로젝트 폴더를 열어 볼 것)
- 읽기 전에 반드시 그 파일을 만드는 코드(또는 직접 만든 txt)가 먼저 실행됐는지 확인
- 메모장으로 데이터 파일을 직접 만들 때: 숫자 사이는 공백이나 줄바꿈이면 충분

### scanf 후 무한 루프 / 입력이 씹힘
- 숫자 입력에 문자가 들어오면 scanf가 실패를 반복함 → 실패 시 버퍼 비우기:
  ```c
  if (scanf("%d", &menu) != 1) { while (getchar() != '\n'); continue; }
  ```
- `%c`로 문자를 받을 땐 `scanf(" %c", &ch);` 처럼 **% 앞에 공백** 넣기 (엔터 문자 무시)

### 0xC0000005 / 프로그램이 갑자기 죽음 (런타임 크래시)
- 거의 항상 **NULL 포인터 접근** 또는 **배열 범위 초과**
- 체크리스트:
  - `fopen` 결과 NULL 검사했나?
  - `bstSearch` 결과가 NULL인데 `->key` 한 것 아닌가?
  - 배열 크기보다 많이 읽거나 쓴 것 아닌가? (`N`, `CQ_SIZE`, `STACK_SIZE` 확인)
  - 공백 큐/스택에서 꺼낸 값을 그대로 쓴 것 아닌가?
- 디버깅 요령: 의심 구간 곳곳에 `printf("여기1\n");` 를 넣고 어디까지 출력되는지 본다

### 한글 출력이 깨짐 (콘솔에서 ???나 외계어)
- 콘솔 인코딩 문제. 채점에는 보통 영향 없지만 신경 쓰이면:
  - main 시작 부분에 `system("chcp 949");` 추가 (#include <stdlib.h> 필요)
  - 그래도 깨지면 출력 문구를 영어로 바꾸는 것이 가장 빠름

## 4. 빌드도 되고 실행도 되는데 결과가 이상함

### 난수가 실행할 때마다 똑같음
- `initRandom();` (= `srand(time(NULL))`)을 main 맨 앞에서 호출 안 함

### 정렬 방향이 반대로 나옴
- 정렬 함수의 `ascending` 인자 확인: **1 = 오름차순, 0 = 내림차순**

### BST에 넣었는데 트리가 비어 있음 / 삭제가 안 됨
- `bstInsert(root, k);` 처럼 반환값을 안 받았기 때문 → **`root = bstInsert(root, k);`**

### 파일에 결과가 안 써짐 / 반만 써짐
- `fclose(fp);` 누락 (버퍼가 파일로 안 내려감). 쓰기가 끝나면 즉시 fclose!
- 같은 파일을 "w"로 두 번 열면 앞 내용이 날아감. 이어 쓰려면 `"a"` 모드

### 큐에 100개를 넣었는데 99개만 들어감
- 원형 큐는 구조상 **한 칸을 비워 두므로 CQ_SIZE-1개**까지 저장됨 → `CQ_SIZE`를 1 크게

### 중복 값을 BST에 넣으면 "이미 같은 키" 메시지
- 정상 동작(BST는 중복 금지). 난수 생성을 `genRandomArrayUnique`로 바꾸면 해결

## 4.5 모듈별 자주 하는 실수 (빌드는 되는데 결과가 이상할 때)

| 모듈 | 흔한 실수 | 바로잡기 |
|---|---|---|
| bst / avl | 삽입·삭제 후 반환값을 안 받음 | **`root = bstInsert(root, k);`** 형태로 받기 |
| search | 이진/색인 탐색인데 정렬을 안 함 | 먼저 `quickSort(a, n, 1);` 등으로 정렬 |
| search | 재귀 이진 탐색 비교횟수가 이상 | 호출 전 **`cmp = 0;`** 으로 초기화 |
| heap | 인덱스 0이 비어 보임 | 힙은 **인덱스 1부터** 사용(정상). 부모=i/2, 자식=2i·2i+1 |
| cQueue / arrayDeque | 크기만큼 안 들어감 | 한 칸을 비워 두는 방식 → 실제 저장은 **크기-1개** |
| bst (treeSort) | 중복 값이 사라짐 | BST는 중복 무시 → 중복 없는 데이터에만 |
| 정렬/큐 공통 | 난수가 매번 같음 | main 맨 앞 **`initRandom();`** 한 번 |
| 구조체 정렬 | sort.h로 구조체가 안 됨 | sort.h는 int 배열 전용 → `sortDronesBy` 패턴 복사 또는 heap의 tag 사용 |
| 큐잉/신뢰성 | 계산값이 -1 | M/M/1은 λ<μ 일 때만 유효(불안정 시 -1), 0으로 나누기 방지값 |

## 5. 최후의 수단 (10분 안에 복구)

1. **프로젝트가 통째로 꼬였을 때**: 새 빈 프로젝트 생성 → 오픈소스 .c/.h 전부 다시 추가 → main.c만 수정 재시작 (5분이면 됨)
2. **어떤 모듈이 의심될 때**: main.c의 해당 `demoXxx()`만 호출해 보면 모듈 자체의 정상 여부를 바로 확인 가능
3. **함수가 기억 안 날 때**: 각 .h 파일의 머리 주석에 사용 예시가 다 있다. **헤더부터 열어 보기**
4. **시간이 없을 때 우선순위**: ① 일단 컴파일되게 만든다 → ② 핵심 로직(정렬/큐/트리)부터 출력 → ③ 파일 출력은 `dualPrintf(log, ...)`로 화면 출력에 끼워 넣으면 한 번에 해결
