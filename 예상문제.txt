문제 1. 은행 번호표 시스템 (원형 큐) - 15점
----------------------------------------------------------------
은행 창구의 대기 번호표를 '원형 큐'로 구현하라.
 (1) 0~50 사이 난수로 "도착 고객 수"를 정하고, 1번부터 차례로 번호표를
     발급(enqueue)하라.
 (2) 창구에서 한 명씩 호출(dequeue)할 때마다 "○○번 고객님 창구로 오세요"를
     화면과 call_log.txt에 동시에 출력하라.
 (3) 원형 큐가 가득 찼을 때와 비었을 때의 메시지를 구분해 출력하라.
 (4) 마지막에 처리한 고객 수와 미처리(대기) 고객 수를 출력하라.
 ※ 배열 크기를 바꿔도 동작하도록 #define으로 큐 크기를 관리할 것.

----------------------------------------------------------------
문제 2. 프린터 작업 대기열 (연결 큐) - 15점
----------------------------------------------------------------
여러 사용자가 보낸 인쇄 작업을 '연결 큐'로 관리하라. 작업은
(작업번호, 페이지 수)로 구성된다.
 (1) 난수로 작업 5~10개를 생성하고 jobs.txt에 "작업번호 페이지수" 형식으로
     저장하라.
 (2) jobs.txt를 읽어 연결 큐에 넣고, 먼저 들어온 순서대로 인쇄하라.
 (3) 한 페이지 인쇄에 1초가 걸린다고 가정하고, 각 작업이 "대기한 시간"을
     누적 계산하여, 모든 작업의 평균 대기 시간을 출력하라.

----------------------------------------------------------------
문제 3. 웹 브라우저 방문 기록 (덱) - 20점
----------------------------------------------------------------
브라우저의 '뒤로/앞으로 가기'를 덱(deque)으로 구현하라.
 (1) 페이지 방문은 덱의 뒤(rear)에 삽입한다.
 (2) '뒤로 가기'는 rear에서 하나 꺼내 따로 보관하고, '앞으로 가기'는
     보관해 둔 것을 다시 rear로 넣는다.
 (3) 방문/뒤로/앞으로 명령을 최소 8개 이상 수행하며 매 단계 현재 페이지와
     덱 상태를 출력하라.
 (4) 덱이 비었을 때 뒤로 가기를 시도하면 "더 이상 뒤로 갈 수 없습니다"를
     출력하라.

----------------------------------------------------------------
문제 4. 콜센터 대기 시뮬레이션 (큐잉 이론) - 20점
----------------------------------------------------------------
콜센터에 전화가 무작위로 걸려오는 상황을 시뮬레이션하라.
 (1) 총 30단위 시간 동안, 매 시간 40% 확률로 전화 1통이 도착한다(난수).
 (2) 상담원은 1명이며, 한 통 상담에 1~4단위 시간이 걸린다(난수).
 (3) 큐에 쌓인 전화는 도착 순서대로 처리한다.
 (4) 시뮬레이션 종료 후 ① 도착한 전화 수 ② 처리한 전화 수
     ③ 평균 대기 시간 ④ 최대 대기열 길이를 출력하라.

----------------------------------------------------------------
문제 5. 수식 괄호 검사 (스택) - 15점
----------------------------------------------------------------
키보드로 입력한 수식 문자열의 괄호 짝이 올바른지 스택으로 검사하라.
 (1) (), {}, [] 세 종류의 괄호를 모두 처리한다.
 (2) 여는 괄호는 push, 닫는 괄호는 pop하여 짝을 비교한다.
 (3) 짝이 맞지 않거나 스택이 비었는데 닫는 괄호가 오면 "불균형"으로 판단.
 (4) 예) "(a+{b-[c]})" -> 균형,  "(a+[b)]" -> 불균형
     서로 다른 입력 3개 이상에 대해 결과를 출력하라.

----------------------------------------------------------------
문제 6. 후위 표기식 계산 (스택) - 20점
----------------------------------------------------------------
후위 표기(postfix)로 주어진 수식을 스택으로 계산하라.
 (1) 피연산자는 한 자리 정수, 연산자는 + - * / 로 한정한다.
 (2) 피연산자는 push, 연산자를 만나면 2개를 pop하여 계산 후 결과를 push.
 (3) 예) "52+8*" -> (5+2)*8 = 56
 (4) postfix.txt에서 수식을 읽어 계산 결과를 result.txt에 저장하라.

문제 7. 이진 트리 순회 3종 (전위/중위/후위) - 20점
----------------------------------------------------------------
다음 수식 트리를 직접 만들고 세 가지 순회 결과를 출력하라.
        식: (A + B) * (C - D)
 (1) 포인터로 트리 노드를 연결하여 위 수식 트리를 구성하라.
 (2) 전위(preorder), 중위(inorder), 후위(postorder) 순회 결과를 각각 출력.
 (3) 중위 순회 결과가 원래 수식과 같은 의미인지 설명을 주석으로 남겨라.
 (4) (심화) 각 순회를 재귀로 구현한 뒤, 같은 결과가 나오는지 확인하라.

----------------------------------------------------------------
문제 8. 영어 단어 사전 (이진 탐색 트리) - 25점
----------------------------------------------------------------
영어 단어들을 이진 탐색 트리에 저장하는 사전 프로그램을 작성하라.
 (1) words.txt에서 단어를 읽어 BST에 삽입한다(알파벳 순 기준).
 (2) 중위 순회로 출력하면 사전 순으로 정렬되어 나와야 한다.
 (3) 사용자가 입력한 단어를 검색하여 있으면 "등재됨", 없으면 "미등재" 출력.
 (4) 특정 단어를 삭제하고(단말/자식1개/자식2개 경우 모두) 다시 중위 순회로
     정렬이 유지되는지 확인하라.
 (5) 트리의 전체 노드 수와 높이를 출력하라.

----------------------------------------------------------------
문제 9. 정수 BST의 통계 (이진 탐색 트리) - 20점
----------------------------------------------------------------
0~999 사이 중복 없는 난수 20개로 이진 탐색 트리를 구성하라.
 (1) 생성한 난수를 nums.txt에 저장한 뒤 읽어서 BST를 만든다.
 (2) 최댓값, 최솟값을 트리 구조를 이용해(전체 탐색 금지) 찾아 출력하라.
 (3) 레벨 순회(큐 사용)로 트리를 위에서 아래로 출력하라.
 (4) 특정 값 X보다 작은 노드의 개수를 출력하라.

----------------------------------------------------------------
문제 10. 스레드 이진 트리 (스택 없는 중위 순회) - 25점
----------------------------------------------------------------
스레드 이진 트리(우 스레드)를 구성하여 스택/재귀 없이 중위 순회하라.
 (1) 정수 7~10개로 이진 탐색 트리를 만든다.
 (2) 단말 노드의 비어 있는 오른쪽 링크를 '중위 후속자'로 연결(스레드)하라.
 (3) 스레드를 따라가며(스택/재귀 사용 금지) 중위 순회 결과를 출력하라.
 (4) 일반 재귀 중위 순회 결과와 동일한지 비교하여 검증하라.

----------------------------------------------------------------
문제 11. AVL 트리의 4가지 회전 (균형 트리) - 30점
----------------------------------------------------------------
AVL 트리를 구현하여 삽입 시 균형이 깨지면 회전으로 복구하라.
 (1) 노드 삽입 후 각 노드의 균형 인수(BF = 왼쪽 높이 - 오른쪽 높이)를
     계산한다.
 (2) 불균형(|BF|>=2) 발생 시 상황에 맞는 회전을 수행하라.
       - LL : 오른쪽 단순 회전
       - RR : 왼쪽 단순 회전
       - LR : 왼쪽-오른쪽 이중 회전
       - RL : 오른쪽-왼쪽 이중 회전
 (3) 다음 두 입력 순서를 각각 삽입하며, 회전이 일어날 때마다 회전 종류와
     그때의 루트를 출력하라.
       (가) 10, 20, 30, 40, 50, 60, 70   (오름차순 -> RR 연쇄)
       (나) 50, 30, 70, 20, 40, 10, 5    (LL / LR 유발)
 (4) 최종 트리의 높이를 같은 값을 넣은 '일반 BST'의 높이와 비교하라.

----------------------------------------------------------------
문제 12. 응급실 우선순위 진료 (힙 / 우선순위 큐) - 25점
----------------------------------------------------------------
응급실 환자를 위급도(1=가장 위급 ~ 5=경미)에 따라 진료하라.
 (1) 환자는 (환자번호, 위급도)로 구성되며, 난수로 8~12명을 생성한다.
 (2) 위급도가 높은(숫자가 작은) 환자를 먼저 꺼내야 하므로 '최소 힙'을 쓴다.
 (3) 같은 위급도 안에서는 먼저 도착한 환자가 먼저 진료받도록 처리 방법을
     설명/구현하라.
 (4) 모든 환자를 진료 순서대로 꺼내 "환자번호(위급도)" 형식으로 출력하라.

----------------------------------------------------------------
문제 13. 최대 히프트리로 상위 K개 뽑기 - 20점
----------------------------------------------------------------
게임 점수 데이터에서 상위 K명을 최대 히프트리로 구하라.
 (1) 0~10000 난수로 점수 30개를 만들어 scores.txt에 저장한다.
 (2) 전체를 최대 힙에 삽입한 뒤, 삭제를 K번 반복해 상위 K개를 얻는다.
 (3) K는 사용자에게 입력받는다(예: 5).
 (4) 상위 K개의 점수와 평균을 출력하라.
